# OpenDomain AI 贡献指南

欢迎你参与OpenDomain AI的社区共建，我们非常感谢每一位开发者的贡献！

## 贡献方式
目前支持以下几种贡献方式，你可以根据自己的能力和时间选择：
**：反馈bug、提出功能需求、优化文档建议
2.  **参与讨论**：讨论项目技术选型、功能设计、落地细节
3.  **提交Pull Request（PR）**：补充代码、完善文档、修复bug（后续版本开放）

##提交问题规范
1.提交前请先搜索已有问题，避免重复提交
2.选择对应的问题模板（bug报告/功能需求）
3.  清晰描述问题/需求，尽可能提供详细信息（如环境、步骤、截图等）

## 后续PR规范（v0.1.0 MVP版后开放）
1.  先Fork本仓库到自己的GitHub账号
2.  克隆Fork后的仓库到本地，创建专属分支（不要直接在main分支开发）
3.  完成开发后，提交清晰的Commit信息，推送到自己的仓库
4.  发起Pull Request，描述清楚修改内容和目的
5.  等待项目维护者审核，配合修改完善

## 代码规范（后续补充）
-  后端Python：遵循PEP 8规范
-  前端Vue3：遵循Vue官方编码规范
-  Commit信息：遵循「类型: 描述」格式（如feat: 新增AI客服上下文记忆功能）
